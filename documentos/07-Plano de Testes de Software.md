# Plano de Testes de Software


Os testes funcionais a serem realizados na aplicação são descritos a seguir.

| Caso de teste | Requisitos Associados | Objetivo do teste | Passos | Critério de exito | Responsável |
| --- | --- | --- | --- | --- | ---- |
| CT 01: Verificar cadastro de usuários | RF: 012 O site deve permitir ao usuário cadastrar uma conta. | Conferir se o cadastro está sendo processado corretamente | 1. Acessar o navegador. 2.Informar o endereço do site. 3. Visualizar a página Home. 4. Clicar em "login", em seguida “novo usuário”. 5. Preencher o formulário e clicar em “submit” | Deve ocorrer uma validação das informações fornecidas pelo usuário, e ao clicar em "Submit", deve aparecer a mensagem "Usuário cadastrado com sucesso" e ser direcionado para página de login. Os dados cadastrados devem ir para o banco de Cadastro de Usuários | Matheus
| CT 02: Verificar login no site | RF: 013 O site deve permitir ao usuário logar no sistema. | Conferir se a função de login funciona corretamente | 1. Um usuário cadastrado deve acessar o navegador.  2. Informarmar os dados de usuário e senha.  3. Clicar em "login".  4. Se o cadastro estiver correto o usuário será encaminhado para página principal, se os dados estiverem incorreto, deve aparecer a mensagem de dados incorretos | O usuário deve ter acesso ao sistema se tiver fornecidos os dados corretos e dados incorretos ou usuáriosnão cadastrados deverão ser avisados  | Fagner
| CT 03: Verificar o botão de relatório de vendas | RF: 08 A aplicação deve ser capaz de gerar um relatório de vendas realizadas. | Conferir se o relatório está sendo gerado corretamente | 1 - Acessar a página de relatórios; 2 – Clicar no botão “Relatório Total de Vendas” 3-Preencher o período solicitado e clicar | Ao clicar no botão deve aparecer na tela o relatório de todas as vendas cadastradas no período solicitado, se não houver vendas deve ser informado ao usuário que nenhuma venda foi efetivada. Caso o usuário desista de gerar o relatório ao fechar o popup ele deve ser redirecionado para a página de relatórios | Fagner
| CT 04: Verificar o botão de relatório de estoque | RF: 09 A aplicação deve ser capaz de gerar um relatório da quantidade total dos produtos em estoque. | Conferir se o relatório está sendo gerado corretamente | 1 - Acessar a página de relatórios; 2 – Clicar no botão “Relatório Total de Estoque”; 3-Preencher o período solicitado e clicar | Ao clicar no botão deve aparecer na tela o relatório de todo o estoque cadastrado no período solicitado, se não houver estoque deve ser informado ao usuário que nenhum produto foi cadastrado. Caso o usuário desista de gerar o relatório ao fechar o popup ele deve ser redirecionado para a página de relatórios | Fagner
|CT 05: Verificar Funcionamento dos links da pagina home.|RF13- A aplicação deve permitir ao usuário fazer o login da sua conta. RF12 - A aplicação deve permitir ao usuário cadastrar uma conta.|Verificar se baseando nos requisitos os links direncionam para a pagina de login e na mesma é possivel fazer login e cadastro.|1.Acessar o Navegador 2.Informar o endereço do site 3.Navegar pela pagina 4.Clicar nos botões 5. Acessar o menu Login 6.Clicar em Cadastar 7.Após cadastrar efetuar login. | Todos os links devem direcionar para seus respectivos locais da pagina. E após todos os procedimentos de cadastro deve ser possivel fazer login com os dados cadastrados.| Mariana
|CT 06: Verificar funcionamento da pagina de estoque |RF-01:A aplicação deve permitir ao usuário fazer o cadastro de itens. RF-02:A aplicação deve permitir ao usuário fazer a consulta do custo dos itens. RF-05:A aplicação deve permitir ao usuário atualizar a quantidade de itens no estoque. RF-07:A aplicação deve permitir ao usuário cadastrar o preço de venda de um item.| Verificar se o cadastro, edição e atualização de itens está fucnionando corretamente! De maneira que o usuário possa realizar o cadastro completo do item, assim com preços e imagens do produto. E após cadastrados fazer uma consulta sob o item.|1.Acessar o Navegador. 2.Informar o endereço do site. 3.Clicar no botão Login 4.Fazer login na aplicação 5.Clicar no botão "Estoque". 6.Clicar no botão "Adicionar". 7.Navegar no modal de cadastro. 8.Preencher dados do produto. 9.Clicar em "finalizar". 10.verificar se o item realmente foi salvo. 11. Clicar no item. 12.Verificar no modal aberto se todas as informações constam lá. 13.Alterar a quantidade em estoque.|Ao clicar em adicionar, a aplicação deve permitir ao usuario cadastrar um item preenchendo todos os seus dados. Após cadastrado o item o usuário deve ser permitido a altera a quantidade em estoque a qualquer momento, assim como deve ser possível também consultar os dados a qualquer momento.| Matheus
 |CT 07: Verificar funcionamento página de estoque - edição e atualização.|RF-06: A aplicação deve permitir ao usuário registrar a compra de um item. RF-11: A aplicação deve permitir ao usuario editar, remover e arquivar algum item do estoque.| Verificar se a aplicação permite ao usuario registrar a compra de um produto alterando automaticamente a quantidade do estoque. E caso o produto cai em desuso que seja possivel inativa-lo.| 1.Acessar o Navegador. 2.Informar o endereço do site. 3.Clicar no botão Login 4.Fazer login na aplicação 5.Clicar no botão "Estoque". 6.Clicar no "item 1". 7.Clicar em "Registrar Compra". 8. Preencher com os dados da compra. 9. Confirmar. 10. Selecionar "item 2". 10. Clicar no botão "lixeira".| Após entrar com os dados da compra a aplicação deve colher os dados dessa compra para consultas futuras, e também deve alterar automaticamente a quantidade de itens em estoque. Após "excluir" o item, ele não deve aparecer mais na lista de estoque. | Matheus
|CT 08: Verificar funcionamento da pagina de vendas.|RF-03:A aplicação deve permitir ao usuário registrar uma venda.| Verificar se a aplicação permite ao usuario registrar a venda de um ou mais produtos de forma ao ser efetivada a venda.|1.Acessar o Navegador. 2.Informar o endereço do site. 3.Clicar no botão Login 4.Fazer login na aplicação 5.Clicar no botão "Vendas". 6. Clicar no "item 1" 7.Selecionar "item 1". 8.Registrar venda do "item 1". 10. Selecionar "item 1", "item 2", "item 3". 11. Registrar venda.|Após selecionar um ou mais produtos, e finalizar a venda! Irá gerar uma venda salva.| Mariana.
